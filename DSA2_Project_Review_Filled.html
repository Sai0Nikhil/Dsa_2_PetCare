2. Project Slides
Abstract PetCare is a Java-based command-line application that models a veterinary clinic and pet health management platform. It manages pet profiles, medical histories, veterinary appointments, vaccination schedules, and clinic analytics using six categories of classical DSA — Trees, Multiway Trees, Graphs, Shortest Paths, Sorting, and Greedy/DP — applied to realistic clinic operations.
Problem Statement Traditional veterinary management systems lack efficient data-structure-backed operations for indexing pet records, optimising routes between clinic branches, and scheduling overlapping appointments. PetCare bridges this gap by mapping each DSA module to a concrete clinical problem domain, demonstrating practical application of all six course modules within a single cohesive system.
Objectives of Project 1. Implement BST & AVL Trees for O(log n) pet record indexing with tree traversals and complexity analysis.
2. Use B-Tree, B+ Tree, Segment Tree & Fenwick Tree for multiway indexing, database-style range queries, and prefix sums.
3. Apply BFS, DFS, connected components, and MST (Kruskal / Prim) for clinic network management.
4. Implement Dijkstra, Bellman-Ford, Floyd-Warshall, and Topological Sort for route optimisation and procedure scheduling.
5. Apply Divide & Conquer paradigm — Merge Sort and Quick Sort — for treatment record ranking.
6. (Planned) Greedy & DP algorithms — Activity Selection, Knapsack, LIS — Coming Soon.
Domain / Application Design Domain: Veterinary Clinic & Pet Health Management
Architecture: Java CLI app with nested menus — one sub-menu per DSA module.
Entities: Pet, Treatment, Appointment, Vaccination, Clinic
Packages: petcare.models, petcare.utils, petcare.menu, petcare.trees, petcare.graphs, petcare.sorting, petcare.algorithms
Selected Data Structures BST, AVL Tree, B-Tree (order t), B+ Tree, Segment Tree, Fenwick Tree, Graph (Adjacency List), Priority Queue, Arrays
Programming Language Used Java (JDK 11+)
Software / Tools Used Eclipse / IntelliJ IDEA / VS Code, Java JDK, Command-Line Interface (CLI), Git
Sample Input/Output Shown Yes — CLI nested menus demonstrate live algorithms on 45 pre-loaded CSV pet records and clinic data.
M1: AVL insert / search / delete, BST traversals, height comparison.
M2: B-Tree search, B+ Tree range query, Segment Tree range sum, Fenwick prefix sum.
M3: BFS hop-distances, DFS connected components, Kruskal & Prim MST.
M4: Dijkstra, Bellman-Ford, Floyd-Warshall distance tables, Topological Sort.
M5: Merge Sort (by cost), Quick Sort (by frequency). Heap Sort & Radix Sort — Coming Soon.
